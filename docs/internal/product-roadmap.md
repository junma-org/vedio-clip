# 产品发展方向

## 定位

本项目优先做本地离线、低学习成本的视频快剪工具。不要直接追求完整剪映替代品，而是覆盖常见批量、秒级、轻量编辑场景。

一句话定位：比剪映简单，比直接写 FFmpeg 好用，本地离线，一分钟完成常见剪辑。

## 模式规划

### 极简模式

- 拖入视频后直接处理。
- 支持剪掉开头、删除指定秒数区间、多个删除区间、调整分辨率。
- 后续可扩展为保留指定区间、批量处理、常用模板。
- UI 保持表单式，不引入复杂时间轴。

### 达人模式

- 目标是轻时间轴，而不是完整专业剪辑软件。
- 进入达人模式前，应先沉淀统一编辑模型，让极简模式和达人模式都能生成同一种编辑描述。
- 第一阶段能力：视频预览、拖动进度、前进/后退一帧、设置入点出点、删除片段、撤销重做、导出。
- 第二阶段能力：手动字幕、SRT 导入、字幕样式、简单文字贴片。
- 第三阶段能力：淡入淡出、静音、模糊背景、马赛克、亮度/音量等模板化效果；效果优先做固定时间段和固定区域模板，不提前承诺复杂跟踪或关键帧动画。

## 技术方向

- 抽象统一的编辑模型，极简模式和达人模式都生成同一种编辑描述。
- FFmpeg 命令构造集中在 `ffmpeg_utils.py`，避免 GUI 和处理逻辑耦合。
- 每次迭代补充内部文档：需求、技术方案、待办、验证结果。

## 版本路线

- v0.2：极简模式增加删除指定秒数区间。代码和自动化测试已完成，真实样片导出验证待补。
- v0.3：支持多个删除区间和区间列表校验，继续保持极简模式表单式 UI。代码和自动化测试已完成，真实样片导出验证待补。
- v0.4：达人模式 MVP 前置版本，抽象统一编辑模型和集中校验，极简模式先接入该模型；暂不新增达人模式界面。代码和自动化测试已完成。
- v0.5：达人模式 MVP，基于统一编辑模型支持预览、打点、删除片段和导出。代码和自动化测试已完成，GUI 手动验证待在 Windows 打包环境补跑。
- v0.6：字幕基础能力，支持达人模式手动字幕、SRT 导入、基础样式和烧录导出。代码和自动化测试已完成，GUI 手动验证待在 Windows 打包环境补跑。
- v0.7：模板化简单特效。
