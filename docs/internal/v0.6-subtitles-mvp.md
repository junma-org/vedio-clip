# v0.6 字幕基础能力

## 需求

- 在达人模式中新增字幕基础能力，覆盖 SRT 导入、手动添加字幕、删除字幕、清空字幕和导出烧录字幕。
- 字幕导出必须复用 `EditPlan`，不要让 GUI 直接拼 FFmpeg 字幕命令。
- 字幕命令构造继续集中在 `ffmpeg_utils.py`。
- 本版本不做字幕轨道、自动识别、逐字字幕、字幕动画、关键帧、多区域贴片或复杂剪映式时间轴。

## 假设和取舍

- v0.6 的“字幕基础能力”定义为硬字幕烧录，导出 MP4 后字幕直接成为画面的一部分。
- 手动字幕以当前达人模式入点/出点作为字幕开始/结束时间，文本由用户输入。
- SRT 导入只支持标准 `.srt` 文本字幕；先支持 UTF-8/UTF-8 BOM/GBK 读取兜底，不支持 ASS 样式导入。
- 字幕样式只做全局基础项：字号和底部边距。颜色、描边和字体先固定为易读默认值。
- 若同时做了删除区间，字幕时间仍按原视频时间写入 FFmpeg filter；FFmpeg 过滤链会和删除片段的 select/setpts 一起工作，真实样片需要补手动验证。

## 技术方案

- 新增 `subtitle_model.py`：
  - `SubtitleEntry` 表示单条字幕。
  - `SubtitleStyle` 表示全局基础样式。
  - `SubtitleTrack` 表示字幕集合和启用状态。
  - 提供 SRT 解析、SRT 序列化、字幕归一化和校验。
- 扩展 `EditPlan`：
  - 增加 `subtitles: SubtitleTrack` 字段。
  - `normalized()` / `validate()` 同步归一化字幕。
- 扩展 `ffmpeg_utils.py`：
  - 为有字幕的 `EditPlan` 创建临时 SRT 文件。
  - 在视频 filter 链中追加 `subtitles=...`，并用 `force_style` 控制基础字号和底部边距。
  - 导出线程完成后清理临时字幕文件。
- 扩展达人模式 GUI：
  - 增加字幕文本输入、字号、底部边距。
  - 使用入点/出点添加手动字幕。
  - 支持导入 SRT、删除选中字幕、清空字幕。
  - 导出时把字幕集合写入 `EditPlan`。

- v0.6 字幕基础能力代码和自动化测试已完成。
- 达人模式支持手动添加字幕、导入 SRT、删除/清空字幕、设置全局字号和底部边距。
- 导出复用 `EditPlan`，字幕由 `ffmpeg_utils.py` 写出临时 SRT 并追加 `subtitles` filter 烧录。

## 遗留风险和下一步

- 需要在安装了 `PySide6` 和 FFmpeg 的 Windows 打包环境中手动验证字幕 GUI、SRT 导入和真实视频烧录。
- 字幕与删除区间同时存在时，当前方案先按原始时间烧录字幕，再做删除区间过滤；自动化只覆盖命令构造，真实样片仍需补验。
- 若后续需要更精细样式，再扩展颜色、描边、字体和位置；本版本不提前做多样式字幕轨道。
