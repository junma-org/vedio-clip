# v0.13 达人模式预览、时间线与性能优化

日期：2026-05-01

## 需求

- 达人模式功能增多后，预览区容易被压得过矮，需要支持用户自由上下拖拽调整预览高度。
- 预览播放区域需要支持独立弹出，便于在大预览窗口中播放和校对。
- 时间线缩放后应有清晰的横向滚动能力，避免播放未结束时因为缩放和自动跟随导致“跳回前面”的感受。
- 极致优化叠加轨拖动性能，尤其是添加视频叠加轨后，点击选区并左右移动时不能导致整窗未响应。
- 优化启动速度，减少首屏初始化阶段的同步阻塞。

## 假设

- 本轮不改变编辑模型和 FFmpeg 导出语义，只优化达人模式 GUI 交互和预览同步策略。
- 叠加视频预览可以在拖拽过程中降级为时间轴预览，松手后再同步播放器位置；这比逐帧同步所有播放器更稳定。
- 弹出预览复用当前 `QGraphicsScene`，不额外创建主播放器，避免双播放器状态不一致。

## 技术方案

- 使用 `QSplitter(Qt.Vertical)` 承载预览区和时间线区，让用户直接拖拽两块区域高度。
- 新增预览弹窗，内部只创建第二个 `PreviewGraphicsView` 指向同一个场景，窗口关闭时释放引用。
- 将时间线横向滚动条移动到时间线下方，并根据缩放状态启用；时间线控件增加滚轮横移和 `Ctrl+滚轮` 缩放信号。
- 增加拖拽预览节流：叠加轨拖动时只更新控件和时间轴，不在每个鼠标移动事件调用 `QMediaPlayer.setPosition()`。
- 预览视图避免 `FullViewportUpdate`，降低图形视图重绘成本。
- 启动阶段延后 FFmpeg 检测，窗口先显示，再用 `QTimer.singleShot(0, ...)` 执行检测。
