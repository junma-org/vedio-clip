# v0.8 字幕与达人模式交互优化

日期：2026-04-27

## 需求

- 修正“短视频口播”字幕位置，避免手机竖屏底部倒计时、声明文字遮挡；目标位置为距离底部约 30%-35%。
- 字幕模板需要有明显差异，不只改变坐标。
- 达人模式字幕支持自由修改字体、字号、颜色，并支持渐隐渐显。
- 字幕时间轴支持拖动选中字幕块与字幕边界；拖动时先更新编辑区数值和预览，最终仍需点击“更新”提交。
- 拖拉时间轴选区或字幕边界时，播放头与视频预览帧同步跟随拖动位置。
- 达人模式支持输出分辨率选择，并在预览右下角即时反映比例变化。
- 清空操作增加二次确认。
- 支持 Ctrl+Z 撤销上一次编辑操作。

## 实现口径

- 字幕样式继续复用现有 ASS 工程模型，不引入新的字幕格式。
- 渐隐渐显通过 ASS leading tag `\fad(in,out)` 实现，导入 ASS 原始标签继续尽量保留。
- 字幕拖动采取“拖动预览 + 更新提交”的交互：拖动只改右侧编辑控件和播放头，不直接改 `SubtitleProject`。
- 撤销先覆盖本次涉及的本地编辑状态：删除区间、字幕工程、字幕选中行、选区和达人模式输出分辨率。
- 达人模式分辨率选择复用现有 `OutputOptions.resolution`，导出时和极简模式一致走 FFmpeg `scale`。

## 预览居中修复 2026-04-27

### 需求

- 横屏视频切换到竖屏预览画布时，视频画面必须在预览画布内垂直居中。
- 导出已正确等比缩放补黑边，本次只修预览显示。

### 实现口径

- 显式居中 `QGraphicsVideoItem`，并把视图对齐、`fitInView` 和 `centerOn` 封装成同一个同步步骤。
- 分辨率切换、原生尺寸变化和窗口 resize 后都重新同步预览视图，避免 Qt 多媒体/视图刷新顺序覆盖几何位置。
- 每次 `fitInView` 前重置 view transform，避免旧缩放矩阵影响居中。
- 同步 view 自身 `sceneRect`，避免 view 与 scene 持有不同可视矩形状态。
- `QGraphicsVideoItem` 每次重算时重置 transform/offset，并改为 `IgnoreAspectRatio`，只填满应用自己计算出的等比显示矩形，避免 Qt 视频项内部二次 letterbox 或锚点偏移。

### 验证记录

- `python3 -m py_compile gui.py`
- `python3 -m unittest discover tests`
- `git diff --check`
- 结果：47 个单元测试通过。
- 当前环境缺少 PySide6 运行环境，预览垂直居中仍需打包后人工点验。

## 风险

- Qt 多媒体预览无法在无图形环境中自动验证，只能做逻辑与单元测试验证；交互需要打包后人工点验。
- ASS `\fad` 在部分播放器预览中可能与最终 FFmpeg 烧录存在细微表现差异，最终以 FFmpeg 导出为准。
